## 1. 产品概述

### 1.1 产品名称

**码经** - CodeBible

### 1.2 产品Slogan

> **你的代码宝典，让面试不再是「背经」**

### 1.3 产品愿景

打造一款革命性的前端面试学习平台，通过创新的"通透毛玻璃"UI设计和基于认知科学的记忆机制，帮助开发者真正理解前端技术原理，而非死记硬背"八股文"。让面试准备变得高效、有趣、且记忆深刻。

### 1.4 核心价值

| 核心价值 | 描述 |
|---------|------|
| **真正理解** | 从"背答案"到"懂原理"，每道题都有深度解析和知识关联 |
| **高效记忆** | 基于艾宾浩斯遗忘曲线的间隔重复算法，让知识真正进入长期记忆 |
| **沉浸式体验** | 通透毛玻璃设计语言，柔和光影、优雅过渡、精致细节 |
| **多模态学习** | 文字详解 + 音视频讲解 + 交互式代码演示 + 知识图谱 |
| **个性化路径** | 智能分析薄弱环节，提供定制化学习计划 |

## 2. 市场调研与竞品分析

### 2.1 市场现状

#### 2.1.1 前端面试题市场痛点

根据市场调研，目前前端面试题学习平台存在以下普遍问题：

| 痛点 | 具体表现 | 用户痛苦指数 |
|-----|---------|-------------|
| **形式单一** | 几乎都是纯文字，缺乏多媒体形式 | ⭐⭐⭐⭐⭐ |
| **死记硬背** | 鼓励"背八股"，而非真正理解原理 | ⭐⭐⭐⭐⭐ |
| **缺少记忆辅助** | 没有利用心理学原理帮助记忆，学了就忘 | ⭐⭐⭐⭐⭐ |
| **UI/UX落后** | 设计风格陈旧，缺乏创新，使用体验差 | ⭐⭐⭐⭐ |
| **缺少互动** | 只是被动阅读，没有互动式学习体验 | ⭐⭐⭐⭐ |
| **缺少复习系统** | 没有基于遗忘曲线的智能复习机制 | ⭐⭐⭐⭐⭐ |
| **内容分散** | 各大平台内容分散，缺乏系统化 | ⭐⭐⭐⭐ |

#### 2.1.2 核心发现

1. **2026年前端面试趋势**：不再是靠背诵"八股文"就能过关，企业考察下沉到语言底层原理、工程化实践、复杂场景问题解决
2. **用户核心需求**：从"会答题"到"会思考"，真正理解技术原理，形成完整知识体系
3. **记忆科学验证**：艾宾浩斯遗忘曲线表明，24小时内会遗忘50%，间隔重复是进入长期记忆的关键

### 3.2 差异化核心设计

#### 3.2.1 视觉差异化：通透毛玻璃设计

**不是普通的毛玻璃，而是精心设计的通透质感与光影层次**

| 设计元素 | 传统毛玻璃 | 通透毛玻璃设计 |
|---------|-----------|---------------|
| 透明度 | 固定模糊值 | 精心调校的模糊值与透明度组合，保证可读性 |
| 光影效果 | 静态高光 | 柔和光晕、悬停时的微妙光影变化 |
| 边角处理 | 固定圆角 | 圆润优雅的圆角，交互时的平滑过渡 |
| 色彩融合 | 简单叠加 | 背景色调与前景的和谐融合 |

**核心视觉特性**：
1. **通透质感**：半透明材料呈现柔和的光影层次，精致而不浮夸
2. **柔和光晕**：鼠标悬停时，元素边缘泛起柔和光晕，优雅而有分寸
3. **光带细节**：工具栏与侧边栏之间有微妙的光影过渡
4. **三种主题**：透明、雾面、极光，支持自定义强调色

**每道题目的内容标准**：

| 内容类型 | 要求 | 示例 |
|---------|------|------|
| **题干** | 清晰表述，可能包含场景描述 | "var、let、const的核心区别与底层实现逻辑是什么？" |
| **考点分析** | 明确考察维度，面试官意图 | "核心考察JS词法环境、执行上下文、作用域链底层实现，而非表面语法差异" |
| **文字详解** | 分层次、结构化，包含原理、场景、坑点 | 作用域规则 → 变量提升 → 重复声明 → 赋值规则 → 底层逻辑 |
| **面试官追问** | 预测可能的深度追问 | "为什么for循环中var定义的i外部能访问？const对象为什么可以修改属性？" |
| **视频讲解** | 5-15分钟，白板/动画演示 | 配合可视化动画讲解执行上下文创建过程 |
| **音频讲解** | 可单独播放，适合通勤 | 提取视频音频，支持倍速 |
| **代码演示** | 可编辑、可运行、有输出 | 展示var/let/const在各种场景下的行为差异 |
| **知识图谱** | 关联相关知识点 | 变量提升 → 执行上下文 → 作用域链 → 闭包 |
| **手写练习** | 模拟面试手写场景 | 实现bind、call、apply，手写Promise等 |

**间隔重复算法（SM-2算法优化版）**：

| 自我评估等级 | 描述 | 下次复习间隔 |
|-------------|------|-------------|
| **5 - 完美回忆** | 能够流畅、准确地完整回答，原理理解透彻 | 间隔 × 2.5 |
| **4 - 正确回忆** | 能够正确回答，但需要稍加思考 | 间隔 × 1.8 |
| **3 - 模糊回忆** | 能够回忆起部分内容，查看提示后能完整回答 | 间隔 × 1.2 |
| **2 - 难以回忆** | 几乎忘记，但看到答案后有熟悉感 | 重置为1天 |
| **1 - 完全遗忘** | 没有任何印象，需要重新学习 | 重置为10分钟 |
| **0 - 全新学习** | 第一次学习该知识点 | 10分钟后第一次复习 |

### 5.2 模块一：学习系统（Learning System）

#### 5.2.1 题库浏览与分类

#### 5.3.2 间隔重复算法（SM-2优化版）

**算法核心逻辑**：

```javascript
/**
 * 间隔重复算法（基于SM-2优化）
 * @param {Object} card - 卡片数据
 * @param {number} quality - 自我评估质量 (0-5)
 * @returns {Object} 更新后的卡片数据
 */
function calculateNextReview(card, quality) {
  let { interval, easeFactor, repetitions } = card;
  
  // 质量低于3，重置学习进度
  if (quality < 3) {
    repetitions = 0;
    interval = 1; // 1天后复习
  } else {
    if (repetitions === 0) {
      interval = 1; // 第一次成功复习后，1天后
    } else if (repetitions === 1) {
      interval = 4; // 第二次成功复习后，4天后
    } else {
      // 后续间隔 = 上次间隔 × 难度系数
      interval = Math.round(interval * easeFactor);
    }
    repetitions++;
  }
  
  // 调整难度系数
  // 公式：EF' = EF + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))
  easeFactor = easeFactor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02));
  
  // 难度系数下限
  if (easeFactor < 1.3) {
    easeFactor = 1.3;
  }
  
  // 质量5的额外奖励
  if (quality === 5) {
    easeFactor += 0.1;
  }
  
  return {
    ...card,
    interval,
    easeFactor,
    repetitions,
    nextReview: Date.now() + interval * 24 * 60 * 60 * 1000,
    lastReview: Date.now(),
    lastQuality: quality
  };
}
```

**算法参数说明**：

| 参数 | 说明 | 默认值 |
|-----|------|--------|
| `interval` | 下次复习间隔（天） | 首次: 1天 |
| `easeFactor` | 难度系数（越大越容易） | 2.5 |
| `repetitions` | 连续成功复习次数 | 0 |
| `quality` | 本次复习质量评分 | 0-5 |

## 6. UI/UX交互与视觉设计规范

### 6.1 设计原则

#### 6.1.1 核心设计理念

**通透毛玻璃 - 精致光影设计哲学**：

> 界面不是静态的容器，而是有温度、有生命力的光影载体。它会柔和地响应你的每一个触碰，呈现优雅的光影变化。

| 设计原则 | 描述 | 实现方式 |
|---------|------|---------|
| **通透感** | 元素呈现柔和的半透明质感 | 精心调校的透明度、模糊值组合 |
| **光影层次** | 通过光影创造深度和层次感 | 多层半透明叠加、动态阴影、渐变 |
| **响应性** | 每个交互都有即时反馈 | 悬停光晕、点击弹性、状态过渡 |
| **流畅性** | 元素自然流畅地过渡 | 平滑过渡动画、弹性形变、视差滚动 |
| **极简性** | 内容为王，UI为内容服务 | 减少干扰元素、聚焦当前任务 |

#### 6.1.2 通透毛玻璃设计语言参考

基于通透毛玻璃的设计语言：

| 特性 | 描述 | 应用场景 |
|-----|------|---------|
| **通透质感** | 半透明材料呈现柔和的光影层次 | 导航栏、侧边栏、弹窗背景 |
| **柔和光晕** | 鼠标悬停时元素边缘泛起柔和光晕 | 按钮、卡片、可交互元素 |
| **光影过渡** | 组件之间微妙的光影过渡 | 工具栏与内容区之间 |
| **弹性反馈** | 交互时的平滑弹性反馈 | 点击效果、拖拽反馈 |

#### 7.3.2 间隔重复算法实现

**SM-2算法核心实现**：

```typescript
interface FlashCard {
  id: string;
  questionId: string;
  userId: string;
  
  interval: number;           // 下次复习间隔（天）
  easeFactor: number;         // 难度系数
  repetitions: number;        // 连续成功复习次数
  
  nextReview: Date;           // 下次复习时间
  lastReview: Date;           // 上次复习时间
  lastQuality: number;        // 上次评分
  
  createdAt: Date;
  updatedAt: Date;
}

/**
 * 计算下次复习时间
 * 基于SM-2算法优化
 */
function calculateNextReview(
  card: FlashCard,
  quality: number
): FlashCard {
  let { interval, easeFactor, repetitions } = card;
  
  if (quality < 3) {
    repetitions = 0;
    interval = 1;
  } else {
    if (repetitions === 0) {
      interval = 1;
    } else if (repetitions === 1) {
      interval = 4;
    } else {
      interval = Math.round(interval * easeFactor);
    }
    repetitions++;
  }
  
  easeFactor = easeFactor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02));
  
  if (easeFactor < 1.3) {
    easeFactor = 1.3;
  }
  
  if (quality === 5) {
    easeFactor += 0.1;
  }
  
  const now = new Date();
  const nextReview = new Date(now.getTime() + interval * 24 * 60 * 60 * 1000);
  
  return {
    ...card,
    interval,
    easeFactor,
    repetitions,
    nextReview,
    lastReview: now,
    lastQuality: quality,
    updatedAt: now,
  };
}

/**
 * 获取今日待复习卡片
 */
function getCardsForReview(
  cards: FlashCard[],
  limit: number = 50
): FlashCard[] {
  const now = new Date();
  
  const dueCards = cards
    .filter(card => card.nextReview <= now)
    .sort((a, b) => {
      if (a.lastQuality < 3 && b.lastQuality >= 3) return -1;
      if (a.lastQuality >= 3 && b.lastQuality < 3) return 1;
      return a.nextReview.getTime() - b.nextReview.getTime();
    });
  
  return dueCards.slice(0, limit);
}
```

/**
 * 安全执行JavaScript代码
 * 使用Web Worker隔离执行环境
 */
class SafeCodeExecutor {
  private worker: Worker | null = null;
  
  constructor() {
    this.initWorker();
  }
  
  private initWorker() {
    const workerCode = `
      self.onmessage = function(e) {
        const { code, timeout = 5000 } = e.data;
        
        try {
          const startTime = Date.now();
          const logs = [];
          const originalConsole = { ...console };
          
          console.log = (...args) => logs.push({ type: 'log', args });
          console.warn = (...args) => logs.push({ type: 'warn', args });
          console.error = (...args) => logs.push({ type: 'error', args });
          
          const result = eval(code);
          
          if (Date.now() - startTime > timeout) {
            throw new Error('Execution timeout');
          }
          
          self.postMessage({
            success: true,
            result: this.safeSerialize(result),
            logs
          });
        } catch (error) {
          self.postMessage({
            success: false,
            error: {
              message: error.message,
              stack: error.stack
            }
          });
        }
      };
      
      function safeSerialize(value) {
        if (value === undefined) return 'undefined';
        if (value === null) return 'null';
        if (typeof value === 'function') return value.toString();
        try {
          return JSON.stringify(value);
        } catch {
          return String(value);
        }
      }
    `;
    
    const blob = new Blob([workerCode], { type: 'application/javascript' });
    this.worker = new Worker(URL.createObjectURL(blob));
  }
  
  async execute(code: string, timeout: number = 5000): Promise<{
    success: boolean;
    result?: string;
    logs?: Array<{ type: string; args: any[] }>;
    error?: { message: string; stack?: string };
  }> {
    return new Promise((resolve, reject) => {
      if (!this.worker) {
        reject(new Error('Worker not initialized'));
        return;
      }
      
      const timer = setTimeout(() => {
        reject(new Error('Execution timeout'));
      }, timeout + 1000);
      
      this.worker.onmessage = (e) => {
        clearTimeout(timer);
        resolve(e.data);
      };
      
      this.worker.onerror = (error) => {
        clearTimeout(timer);
        reject(error);
      };
      
      this.worker.postMessage({ code, timeout });
    });
  }
  
  destroy() {
    if (this.worker) {
      this.worker.terminate();
      this.worker = null;
    }
  }
}

## 10. 风险评估与应对策略

### 10.1 技术风险

| 风险 | 可能性 | 影响 | 应对策略 |
|-----|--------|------|---------|
| **Liquid Glass效果性能问题** | 中 | 高 | 使用CSS硬件加速，低端设备降级效果，提供性能设置选项 |
| **代码执行安全问题** | 中 | 高 | 使用Web Worker隔离执行，超时限制，API白名单，禁止危险操作 |
| **音视频播放兼容性** | 中 | 中 | 使用HLS/MP4兼容格式，提供降级方案，测试主流浏览器 |
| **PWA离线功能复杂** | 中 | 中 | 使用Workbox简化开发，优先实现核心缓存，后续迭代完善 |
| **数据库查询性能** | 低 | 高 | 合理设计索引，使用Redis缓存，考虑分库分表策略 |

### 10.2 产品风险

| 风险 | 可能性 | 影响 | 应对策略 |
|-----|--------|------|---------|
| **内容质量不达标** | 高 | 高 | 建立内容审核机制，邀请专家评审，用户反馈持续优化 |
| **记忆算法效果不佳** | 中 | 高 | A/B测试不同算法，收集用户数据持续迭代，提供个性化参数调整 |
| **用户留存低** | 高 | 中 | 建立激励机制（成就、徽章），社交功能，持续内容更新 |
| **竞争激烈** | 高 | 中 | 突出差异化（Liquid UI、记忆系统、多模态内容），快速迭代 |

### 10.3 运营风险

| 风险 | 可能性 | 影响 | 应对策略 |
|-----|--------|------|---------|
| **用户增长缓慢** | 高 | 中 | 社区运营、内容营销、SEO优化、邀请奖励机制 |
| **内容版权问题** | 中 | 高 | 原创内容为主，合理使用授权内容，建立版权审核流程 |
| **用户数据安全** | 低 | 高 | 端到端加密，定期安全审计，合规认证，数据备份 |
