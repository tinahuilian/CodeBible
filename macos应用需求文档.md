# macOS应用需求文档

## 项目概述

### 1. 项目名称
* （待确定，建议：LiquidCare 或 轻伴）

### 2. 项目目标
打造一款具有情感陪伴价值的macOS日常关怀应用，通过创新的液态半透明UI设计和人性化交互，帮助用户建立健康的生活习惯，同时提供温暖、贴心的陪伴感。

### 3. 核心价值
- **情感陪伴**：像一个温暖的朋友，而非冰冷的提醒工具
- **创新设计**：采用macOS最新液态UI设计语言，半透明、流动感、现代感
- **极简交互**：通过全局快捷键快速调用，不打断用户工作流
- **个性化关怀**：根据用户行为和偏好，提供定制化的提醒内容

## 核心功能模块

### 模块1：快捷启动与液态UI界面

#### 1.1 全局快捷键调用
- **功能描述**：通过macOS系统级快捷键 `Cmd + L` 快速调出应用界面
- **技术实现**：使用 `NSEvent` 或 `MASShortcut` 框架实现全局热键监听
- **交互细节**：
  - 按下快捷键后，界面从屏幕底部平滑上浮
  - 释放快捷键后，若未进行交互，界面自动收回
  - 动画效果采用液体流动的物理效果模拟

#### 1.2 液态半透明UI设计
- **视觉设计**：
  - 采用macOS最新液态玻璃态（Glassmorphism）设计
  - 背景为半透明磨砂玻璃效果，配合适当的模糊和亮度
  - 界面边缘圆润，带有微妙的光泽和高光效果
  - 整体色调采用温暖的渐变色彩，根据时间段自动调整
- **交互效果**：
  - 鼠标悬停时，界面有轻微的"波动"效果
  - 点击按钮时，有"挤压"和"反弹"的物理反馈
  - 界面展开和收起时，有液体流动的动画效果

### 模块2：智能提醒系统

#### 2.1 时间段划分与提醒内容
- **白天模式（8:00 - 20:00）**：
  - **吃饭提醒**：
    - 触发时间：早餐(8:00-9:00)、午餐(12:00-13:00)、晚餐(18:00-19:00)
    - 提醒内容：根据时间和用户历史行为智能生成
    - 示例："该吃午饭啦，记得享受一顿美味的午餐哦~"
  - **喝水提醒**：
    - 触发时间：每2小时一次（可自定义）
    - 提醒内容：温和的喝水提示
    - 示例："工作累了吗？休息一下，喝杯水吧！"

- **晚间模式（20:00 - 次日8:00）**：
  - **睡眠提醒**：
    - 触发时间：22:00开始，每30分钟一次（可自定义）
    - 提醒内容：根据时间递进，从温和提示到强烈建议
    - 示例1（22:00）："时间不早了，准备休息吧，充足的睡眠很重要哦~"
    - 示例2（23:00）："还在忙吗？记得明天还有重要的事情，早点睡吧~"
    - 示例3（00:00）："夜深了，该好好休息了，睡够8小时才能精神饱满！"

#### 2.2 智能提醒逻辑
- **基于场景的提醒**：
  - 分析用户当前活动状态（通过屏幕活跃时间、应用使用情况判断）
  - 在用户专注工作时，延迟提醒或使用更温和的提示方式
  - 在用户休息或切换应用时，及时弹出提醒
- **渐进式提醒**：
  - 首次提醒：温和的视觉提示+轻微的音效
  - 多次忽略后：更明显的视觉提示+更温暖的文案
  - 长期忽略：记录用户行为，后续调整提醒策略
- **个性化学习**：
  - 记录用户对提醒的响应时间和行为
  - 分析用户偏好，自动调整提醒时间和内容风格
  - 学习用户的作息规律，提供更精准的提醒服务

### 模块3：情感陪伴系统

#### 3.1 个性化角色
- **功能描述**：创建一个虚拟陪伴角色，根据用户偏好和行为展现不同的"性格"
- **角色类型**：
  - **温柔体贴型**：语气温和，关怀备至
  - **活泼开朗型**：语气积极，充满活力
  - **理性智慧型**：语气平静，提供实用建议
- **角色表现**：
  - 通过文案风格体现角色性格
  - 界面表情和动画效果配合角色风格
  - 用户可随时切换角色类型

#### 3.2 情绪感知与回应
- **功能描述**：通过用户的交互行为和时间背景，感知用户的情绪状态并提供相应的陪伴
- **实现方式**：
  - 分析用户点击提醒的速度和频率
  - 观察用户使用应用的时间和习惯
  - 结合时间因素（如工作日vs周末，白天vs夜晚）
- **回应策略**：
  - 用户快速忽略提醒：可能在忙碌，提供更简短、不打扰的提醒
  - 用户多次查看提醒但未行动：可能有困扰，提供更体贴的关怀
  - 用户深夜还在使用：可能失眠或加班，提供温暖的陪伴和鼓励

#### 3.3 互动式对话
- **功能描述**：提供简单的互动式对话功能，让用户可以与应用进行简短交流
- **交互方式**：
  - 点击界面上的"聊一聊"按钮，展开简短的对话界面
  - 提供预设的情感表达选项（如"我今天很累"、"我很开心"、"我需要鼓励"等）
  - 应用根据用户选择提供相应的回应
- **对话内容**：
  - 基于预设的情感模板生成
  - 结合用户历史行为和当前场景
  - 保持简短、温暖、有针对性

### 模块4：数据统计与个性化设置

#### 4.1 习惯追踪与统计
- **功能描述**：记录用户的健康习惯数据，提供可视化统计和分析
- **统计内容**：
  - 喝水记录：每日喝水次数、时间段分布
  - 吃饭记录：每日三餐的响应时间
  - 睡眠记录：每日睡眠时长、入睡时间
- **数据展示**：
  - 简洁的图表展示（柱状图、折线图）
  - 每日/每周/每月的统计报告
  - 习惯养成进度追踪

#### 4.2 个性化设置
- **提醒设置**：
  - 自定义提醒时间和频率
  - 调整提醒强度和方式
  - 设置免打扰时间段
- **界面设置**：
  - 自定义主题颜色
  - 调整界面透明度和模糊度
  - 选择角色类型和语言风格
- **数据管理**：
  - 数据导出和导入
  - 隐私设置（是否启用数据统计）
  - 重置所有设置

## 技术架构

### 1. 技术栈
- **开发语言**：Swift 5.9+
- **框架**：
  - AppKit（macOS原生UI）
  - Core Data（数据持久化）
  - UserNotifications（系统通知）
  - Combine（响应式编程）
- **第三方库**：
  - MASShortcut（全局快捷键）
  - Lottie（动画效果）
  - Charts（数据可视化）

### 2. 架构模式
- 采用MVVM架构，结合SwiftUI和AppKit
- 使用Core Data进行本地数据存储
- 实现依赖注入，便于测试和维护

### 3. 核心模块划分
1. **App层**：应用生命周期管理、全局状态
2. **Feature层**：各功能模块（快捷启动、提醒、陪伴、统计）
3. **Core层**：通用工具、数据模型、服务层
4. **Infrastructure层**：数据库、网络、系统服务

## UI/UX设计规范

### 1. 设计原则
- **极简主义**：界面简洁，只展示必要信息
- **情感化设计**：通过色彩、动画、文案传递温暖感
- **一致性**：遵循macOS设计规范，保持体验一致
- **可访问性**：确保所有用户都能方便使用

### 2. 色彩规范
- **主色调**：温暖的渐变色彩，根据时间段自动调整
  - 白天：柔和的橙色、黄色系
  - 夜晚：柔和的蓝色、紫色系
- **辅助色**：与主色调协调的中性色
- **文字色**：高对比度，确保可读性

### 3. 交互设计
- **手势**：支持鼠标悬停、点击、拖拽等基本操作
- **动画**：所有交互都有平滑的过渡动画
- **反馈**：每个操作都有明确的视觉和音效反馈

## 差异化创新点

### 1. 液态UI交互创新
- 不仅仅是视觉上的半透明效果，而是完整的液态物理交互
- 模拟液体流动、挤压、反弹等物理效果
- 创造出独特、有趣的用户体验

### 2. 情感陪伴差异化
- 从"提醒工具"转变为"情感陪伴者"
- 通过个性化角色和情绪感知，建立真正的情感连接
- 不是简单的任务提醒，而是温暖的生活伙伴

### 3. 智能提醒创新
- 基于用户行为和场景的智能分析
- 渐进式提醒策略，避免打扰用户
- 个性化学习，不断适应用户习惯

### 4. 极简设计哲学
- 全局快捷键调用，不占用 dock 栏空间
- 界面极简，只展示必要信息
- 不打断用户工作流，真正融入日常使用

## 风险评估与应对策略

### 1. 技术风险
- **风险**：液态UI动画效果可能影响性能
- **应对**：优化动画实现，使用硬件加速，确保在不同配置的mac上都能流畅运行

### 2. 用户接受度风险
- **风险**：用户可能不习惯这种创新的交互方式
- **应对**：提供详细的使用说明，保留传统提醒方式作为可选，收集用户反馈持续优化

### 3. 隐私风险
- **风险**：收集用户行为数据可能涉及隐私问题
- **应对**：
  - 所有数据本地存储，不上传服务器
  - 提供数据管理和删除功能
  - 明确的隐私政策和用户同意机制
